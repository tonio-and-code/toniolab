# English Learning System -- Open Source Strategy

## Draft v0.1 -- 2026-02-14

---

## What This Is

英語学習の「型」を作った。不完全だけどオリジナル。

- ネイティブの話し方を構造的に分析して（Journal #110, #111, #112）
- そこから「本物の英語はこう聞こえる」というスペックを作って（memoria-speech-spec.md, word-review-spec.md）
- そのスペックに基づいた学習システムを一人で組んだ

TOEICでもDuolingoでもPROGRITでもない。「ネイティブが実際にどう喋ってるか」から逆算した英語学習。

---

## The Problem

日本の英語学習市場はこうなってる：

| カテゴリ | 代表例 | 問題 |
|---------|--------|------|
| 試験対策 | TOEIC本、英検 | テストはできるけど喋れない |
| アプリ | Duolingo, スタサプ | ゲーミフィケーションで続くけど実力つかない |
| コーチング | PROGRIT (687,500円/3ヶ月) | 会話レッスンなしで68万。コーチが「勉強しろ」って言うだけ |
| 英会話 | 駅前留学系 | 週1回30分で何が変わるのか |
| YouTube | 大量 | 「ネイティブはこう言う！」系。断片的。体系がない |

**空いてるもの：**
- ネイティブの発話構造を分析して、それに基づいた体系的な学習システム
- 「実際の英語はこう聞こえる」から始まる教材
- note.comのスピーキング構造ニッチ（#113で確認済み。DR92、ほぼ空き）

## What I Built

### System Architecture

```
Word Review（単語復習カレンダー）
  - 50語 = 1シナリオ = 5日分
  - 各日10語 x キャラクターが喋るストーリー形式
  - マスタリーシステム（0=未, 1, 2, 3=済）
  - プロローグ（登場人物・舞台設定）付き

Memoria（会話リスニング）
  - シナリオごとに5日分の会話
  - EN/JP並列表示 + TTS再生
  - 行ごとのディープリンク
  - 話者別カラーリング

Expressions（表現集）
  - シナリオごとに75表現
  - Memoriaの該当行へディープリンク
  - DB連携で進捗管理

US Map（アメリカ発見マップ）
  - 50州 + DC、TopoJSON地図
  - 州ごとに5表現、チェックボックスマスタリー
  - ランクシステム（Newcomer → All-American）

Speaking Guide（スピーキングガイド）
  - ネイティブ発話分析に基づく6ステップメソッド
  - Rojasモデル（clarity > speed）
  - 会話サバイバルキット
```

### The Specs（これが本体）

**memoria-speech-spec.md** -- ネイティブ英語の深層構造
1. Information Density: 発話の70%はスキャフォールディング（意味のある単語は30%だけ）
2. Cognitive Markers: 5-8行ごとにum, uh, false startが必要
3. Clause Chain: 30%以上の発話が4+ clauseをadhesive connectorで接着
4. g-dropping: 70-80%（100%は不自然）
5. Turn Length: 20%が短い(1-5語)、20%が長い(50語+)
6. Repetition: 重要ポイントは2-3回言い換える
7. Cross-Turn Building: 20%以上のターンが前の話者を参照

**word-review-spec.md** -- 復習文のルール
- Speaking-only。ナレーション禁止
- Ultra-casual spoken English
- 50文 = 1つのストーリー。ランダム文禁止
- キャラクターが実際に喋ってる形式

これらのスペックが、市場に存在しない価値。コードは手段。スペックが思想。

## Open Source Philosophy

### 何を公開するか

| 公開するもの | 理由 |
|-------------|------|
| システムのコード全体 | 型として使ってほしい。カスタマイズ自由 |
| Speech Specs | この思想を広めることが目的 |
| システムアーキテクチャ | 英語学習アプリを作る人の参考に |
| サンプルシナリオ | 動作確認用。2-3シナリオ分 |

### 何を公開しないか（要検討）

| 非公開候補 | 理由 |
|-----------|------|
| 全シナリオのコンテンツ | 手作業で書いたオリジナルストーリー |
| ジャーナル本文 | 個人の思想・経験 |
| used-idioms.json全体 | 15,000イディオムの進捗管理 |

### 思想

> 不完全だけどオリジナルな型を用意した。
> あとは好きにやってもらっても構わないし、一緒にやってくれると嬉しい。

- 完璧を目指さない。不完全なまま出す
- バグは一人で直さない。みんなで直す
- カスタマイズ自由。フォーク自由
- コントリビューター歓迎。でも強制しない

## Evangelist Strategy（布教戦略）

### Phase 1: 思想を伝える（無料）

**Where:** note.com（DR92、スピーキング構造ニッチが空いてる）

**What:** スピーキングガイドの内容を記事シリーズとして無料公開
1. 「なぜ日本人は英語が喋れないのか -- ネイティブの発話構造を分析してわかったこと」
2. 「ネイティブは発話の70%をムダに使ってる -- 英語の本当のリズム」
3. 「RojasモデルとAdyashantiモデル -- clarity > speed の英語術」
4. 「会話サバイバル -- 相槌だけで3分持たせる技術」
5. 「英語学習システムをオープンソースで作った話」

**Goal:** 「この人の分析面白い」と思う人を集める。売らない。

### Phase 2: コードを公開する

**Where:** GitHub

**What:**
- リポジトリ公開（README整備）
- Speech Specsをドキュメントとして独立させる
- Getting Started（ローカルで動かす手順）
- Contributing Guide

**Goal:** 開発者・英語教育者が「これ使いたい」と思う状態。

### Phase 3: コミュニティ（自然発生を待つ）

- note.comの読者とGitHubのコントリビューターが重なり始める
- Discord or GitHub Discussionsで会話が始まる
- シナリオを書く人、バグを直す人、翻訳する人が出てくる
- **出てこなくても問題ない。一人でも続ける。**

### Phase 4: 収益化（結果的に）

- note.comメンバーシップ（月500円くらい。有料限定のシナリオ解説）
- GitHub Sponsors
- プレミアムシナリオパック（全コンテンツ含む）
- コンサルティング・ワークショップ（依頼があれば）

**重要: Phase 4は急がない。Phase 1-2を半年やってから考える。**

## Competitive Advantage（なぜこれが勝てるか）

1. **分析の深さ**: Journal #110-112のレベルで英語の発話構造を分析してる人が日本にいない
2. **スペックの存在**: 「ネイティブの英語はこう聞こえるべき」を7つのルールで定義してる教材が存在しない
3. **実装済み**: 分析だけじゃなくて動くシステムがある
4. **思想がある**: 「なぜこう作ったか」のストーリーがジャーナル44本分ある
5. **オープンソース**: 英語学習アプリでOSSは珍しい。差別化になる

## Risks（正直に）

- 営業ができない。人に勧められない
- 「いっつもなにもできない」パターンが繰り返される可能性
- オープンソースにしても誰も来ない可能性
- note.comに書いても読まれない可能性
- 英語学習市場はレッドオーシャン（ただしスピーキング構造ニッチは青い）
